# IDE Editing Primitives Design

> Sub-project of lorien IDE. Adds the missing "build a workflow from scratch" primitives.

## Goal

The lorien workflow editor can currently render and edit existing workflows but can't:

- Add new nodes
- Delete nodes
- Disconnect existing edges easily
- Inspect what a selected node looks like in detail

This spec covers all four. After this ships, a user can build a workflow start-to-finish from the IDE without touching the JSON.

## Scope

### In scope

- Add Node, via three discoverable affordances:
  - `Ctrl/Cmd+K` command palette
  - Right-click empty canvas → context menu (search + node list + "New custom node")
  - Drag a `.ts` node file from the left files sidebar → drop on canvas
- "New custom node" creates a new `.ts` file in `nodes/` from a `defineNode` template, then adds an instance referencing it
- Delete selected node (Backspace/Delete) — also cleans up `in:` references in other nodes that pointed at it
- Delete edge (Backspace/Delete on selected edge, or grab an edge endpoint and drop on empty canvas)
- Inspector panel populated when a node is selected — sectioned stack layout

### Out of scope (deferred)

- Editable config form inside Inspector — read-only JSON for v1
- Rename node id inline — deferred; user can edit JSON
- Node-level right-click menu (duplicate, copy, etc.)
- Type validation on connect (any → any allowed today; runtime catches)

## Architecture

### Selection store (new)

`packages/ide/src/store/selection.ts` — Zustand store:

```ts
interface SelectionState {
  selectedNodeId: string | null
  setSelected: (id: string | null) => void
}
```

The workflow editor sets `selectedNodeId` on `onNodeClick` and clears it on `onPaneClick`. Inspector reads from it. State resets when the active workflow tab changes (subscribe to `useTabsStore`'s `activeWorkflowId`).

### Add Node — three entry points, one component

A single `<AddNodePalette>` component encapsulates the searchable node list. Three triggers consume it:

1. **Command palette (Ctrl+K):** Global keyboard listener; opens a centered shadcn `Dialog` containing `<AddNodePalette mode="palette" onPick={addAtViewportCenter} />`.
2. **Right-click context menu:** React Flow's `onPaneContextMenu` opens a positioned `<Popover>` with the same palette plus a "+ New custom node" button. Position: at the right-click coordinates (translated to flow coordinates for node placement).
3. **Drag from sidebar:** The file tree marks `.ts` files under `nodes/` as `draggable`. On drag-start, `dataTransfer.setData("application/lorien-node", relativePath)`. The canvas's `onDrop`/`onDragOver` handlers read it and add a node at the drop position.

The new node's `uses` field is the schema key (e.g., `@core/response` or `./nodes/users/save-user`). The new id is generated by taking the last segment of `uses`, slugified, with an integer suffix if collision (e.g., `save-user`, `save-user-2`).

### "New custom node" template

When the user clicks "+ New custom node" in the right-click menu:

1. Modal asks for a file path (suggested default: `nodes/<unnamed>.ts`)
2. POST `/api/workspace/file` with a starter template:
   ```ts
   import { defineNode } from "@darrylondil/lorien-runtime"
   import { z } from "zod"
   
   export default defineNode({
     inputs: z.object({}),
     outputs: z.object({}),
     async run(input) {
       return {}
     },
   })
   ```
3. After write succeeds, the schemas endpoint is invalidated and re-fetched
4. The new node is added to the workflow at the right-click position

The PUT endpoint already exists; add a "do-not-overwrite" guard so we don't clobber an existing file. New endpoint or guard flag.

### Delete

- React Flow's `onNodesDelete(deletedNodes)`:
  - Remove from `workflow.nodes`
  - Walk every remaining node's `in:` block; remove any entry whose reference's nodeId matches a deleted id (handles both `in: string` and `in: { ... }` forms)
- React Flow's `onEdgesDelete(deletedEdges)`:
  - Each edge in the rendered DOM is a merged group of N underlying mappings (from the previous work). For each underlying mapping in the deleted edge's `data.mappings`, remove the corresponding entry from `workflow.nodes[target].in`. If the target was `in: string` form, clear it.

### Disconnect by drag-to-empty

- React Flow exposes `onReconnectStart`, `onReconnect`, `onReconnectEnd`:
  - `onReconnect(oldEdge, newConnection)`: user re-targeted the edge — update `in:` accordingly (treated as delete + add)
  - `onReconnectEnd(event, edge, isConnected)`: if `isConnected === false`, the user dropped on empty canvas — delete the edge (same as `onEdgesDelete` path)

### Inspector panel content

`packages/ide/src/inspector/inspector-panel.tsx` — sectioned stack (single column scrollable):

1. **Node section:** id, uses, color swatch (if `defineNode.color` set)
2. **Inputs section:** the inputs JSON Schema rendered as a labeled tree. Each property: name + type. Indented nested objects.
3. **Outputs section:** same shape as Inputs but for outputs.
4. **Config section:** read-only JSON of the node's `config` block. Empty state `(none)` if missing.

Each section has a small uppercase label header (matching the existing IDE style). No collapsibles for v1.

Empty state when no node selected: `(no selection)` text centered.

## Data flow

```
Files panel  ──drag .ts──┐
                          │
Ctrl+K       ──open────┐  │
                       ▼  ▼
Right-click  ──open──► AddNodePalette ──pick──► addNode(workflow, uses, position)
                                                       │
                                                       ▼
                                              workflow.nodes[id] = {...}
                                              workflow.view[id]  = {x, y}
                                              dirty = true

Click node ───────────────► selectionStore.setSelected(id) ───► Inspector renders
Click pane ───────────────► selectionStore.setSelected(null)

Backspace on node ────► onNodesDelete ────► remove node + cleanup refs
Backspace on edge ────► onEdgesDelete ────► remove referenced `in:` entries
Drag edge end to empty ─► onReconnectEnd(connected=false) ─► same as edge delete
```

## Error handling

- Add Node duplicate id: increment suffix until unique. Always succeeds.
- New custom node file already exists: backend returns 409; modal shows "File already exists". User must pick a different path or cancel.
- Schemas not yet loaded when palette opens: show "(loading…)" placeholder.
- Delete a node referenced by other nodes: silently strip the references (don't block). User sees those nodes' inputs become unsatisfied (which the existing smart-expansion already handles by expanding the input branch).

## Testing

- **Add Node:** unit tests for `addNode(workflow, uses, position)` — generates unique id, sets view, marks dirty. Integration test in `workflow-editor.test.tsx`: simulate command palette pick → assert node appears.
- **Right-click menu:** integration test simulating `contextmenu` event → menu renders → click an item → node added at click coords.
- **Drag from sidebar:** unit test for the drag/drop handlers; verify dataTransfer contract.
- **New custom node:** test that POST creates the file with the template content, and the schemas refetch picks it up.
- **Delete:** integration test that deleting a node also strips downstream `in:` refs.
- **Disconnect by drag-to-empty:** integration test simulating `onReconnectEnd` with `isConnected=false` → assert edge gone and `in:` entry removed.
- **Inspector:** unit + integration tests: selection store wired, inspector renders correct content for a known fixture node, empty state when no selection.
